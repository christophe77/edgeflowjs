# Internationalization (i18n)

This guide explains how to add multi-language support to EdgeFlow kiosks using `@edgeflow/i18n`.

## Purpose

Industrial kiosks often need to support multiple languages for different regions. The i18n layer provides:

- Locale management (e.g. `en`, `fr`)
- Translation lookup with fallback to default locale
- Param substitution (e.g. `"Hello, {{name}}"`)
- Date and number formatting via `Intl`
- React integration (Provider, hooks)
- **Offline:** All translations are bundled at build time — no network required

## Install

```bash
pnpm add @edgeflow/i18n
```

In a monorepo workspace:

```json
"dependencies": { "@edgeflow/i18n": "workspace:*" }
```

## Setup

### 1. Create translation files

Create JSON files in `src/locales/` (or your preferred structure):

**en.json** (default):

```json
{
  "screen.idle.ready": "Ready",
  "screen.idle.tapToStart": "Tap to start",
  "screen.idle.start": "Start",
  "screen.idle.connected": "Connected",
  "screen.idle.connecting": "Connecting…",
  "nav.maintenance": "Maintenance"
}
```

**fr.json**:

```json
{
  "screen.idle.ready": "Prêt",
  "screen.idle.tapToStart": "Appuyez pour commencer",
  "screen.idle.start": "Démarrer",
  "screen.idle.connected": "Connecté",
  "screen.idle.connecting": "Connexion…",
  "nav.maintenance": "Maintenance"
}
```

Keys use dot notation (`screen.idle.ready`). Flat keys are supported.

### 2. Create the i18n instance

```ts
import { createI18n } from "@edgeflow/i18n";
import en from "./locales/en.json";
import fr from "./locales/fr.json";

const i18n = createI18n({
  defaultLocale: "en",
  translations: {
    en: en as Record<string, string>,
    fr: fr as Record<string, string>,
  },
});
```

### 3. Wrap your app with I18nProvider

```tsx
import { I18nProvider } from "@edgeflow/i18n/react";

ReactDOM.createRoot(root).render(
  <I18nProvider i18n={i18n}>
    <App />
  </I18nProvider>
);
```

## React usage

### useT()

Use the `useT()` hook to get the translation function:

```tsx
import { useT } from "@edgeflow/i18n/react";

function IdleScreen() {
  const t = useT();
  return (
    <div>
      <h1>{t("screen.idle.ready")}</h1>
      <p>{t("screen.idle.tapToStart")}</p>
      <button>{t("screen.idle.start")}</button>
    </div>
  );
}
```

### useI18n()

Access the full i18n instance for locale switching or formatting:

```tsx
import { useI18n } from "@edgeflow/i18n/react";

function LocaleSwitcher() {
  const i18n = useI18n();
  return (
    <div>
      <button onClick={() => i18n.setLocale("en")}>EN</button>
      <button onClick={() => i18n.setLocale("fr")}>FR</button>
    </div>
  );
}
```

## Adding a locale

1. Create `locales/xx.json` with your translations.
2. Import it and add to the `translations` object:

```ts
import xx from "./locales/xx.json";

createI18n({
  defaultLocale: "en",
  translations: { en, fr, xx },
});
```

3. Call `i18n.setLocale("xx")` when the user selects the new locale.

## Param substitution

Use `{{param}}` placeholders in translations:

```json
{
  "greeting": "Hello, {{name}}!"
}
```

```tsx
t("greeting", { name: "User" }); // "Hello, User!"
```

## Formatting

`formatDate` and `formatNumber` use the built-in `Intl` API with the current locale:

```tsx
const i18n = useI18n();
i18n.formatDate(new Date()); // "2/27/2025" (en) or "27/02/2025" (fr)
i18n.formatNumber(1234.56);   // "1,234.56" (en) or "1 234,56" (fr)
```

## Offline behavior

All translations are bundled at build time. No runtime network requests are needed. This is ideal for kiosks that may run offline.
